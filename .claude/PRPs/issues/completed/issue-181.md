## Problem Statement

Thumbnails are successfully generated by ThumbnailService and stored in `backend/data/temp/thumbnails/`, but VideoService.findThumbnail() searches for thumbnails in `backend/data/videos/` (the video file's directory). This directory mismatch causes all video metadata to have `thumbnail: null`, resulting in no thumbnail images displayed in the frontend.

---

## Analysis

### Root Cause / Change Rationale

**5 Whys Analysis:**

WHY: Video thumbnails are not displayed in the UI?
→ BECAUSE: VideoService.findThumbnail() returns null for all videos
→ Evidence: `videoService.ts:197-227` - findThumbnail() only searches in `videoDir` (video's parent directory)

WHY: findThumbnail() returns null even though thumbnails exist?
→ BECAUSE: Thumbnails are stored in `data/temp/thumbnails/` but findThumbnail() searches in `data/videos/`
→ Evidence: `thumbnailService.ts:58` - thumbnails saved to `this.thumbnailsDirectory` (which is `config.thumbnailsDir`)
→ Evidence: `config.ts:55` - `thumbnailsDir = path.join(tempDir, 'thumbnails')` = `data/temp/thumbnails/`

WHY: ThumbnailService stores thumbnails in temp directory?
→ BECAUSE: This is intentional - thumbnails are considered temporary/generated content
→ Evidence: `config.ts:47` - `tempDir = path.join(process.cwd(), 'data', 'temp')`

WHY: VideoService doesn't search in the thumbnails directory?
→ BECAUSE: The original implementation assumed thumbnails would be co-located with videos
→ Evidence: `videoService.ts:198` - `const videoDir = path.dirname(videoPath)` - searches only in video's directory

### Evidence Chain

```
SYMPTOM: No thumbnails displayed in frontend
↓
CAUSE: VideoService.findThumbnail() returns null
↓
ROOT CAUSE: Directory mismatch
  - Generated: config.thumbnailsDir = "data/temp/thumbnails/video.jpg"
  - Searched:  videoDir = "data/videos/video.jpg" (doesn't exist)
```

### Step 2: Update findThumbnail() to check thumbnails directory as fallback

**File**: `backend/src/services/videoService.ts`
**Lines**: 197-227
**Action**: UPDATE

**Current code:**

```typescript
private async findThumbnail(videoPath: string): Promise<string | null> {
  const videoDir = path.dirname(videoPath);
  const baseName = path.basename(videoPath, path.extname(videoPath));
  
  for (const ext of this.THUMBNAIL_EXTENSIONS) {
    const exactThumbPath = path.join(videoDir, baseName + ext);
    try {
      await fs.access(exactThumbPath);
      return baseName + ext;
    } catch {
      // Exact match failed, continue to case-insensitive fallback
    }
  }

  try {
    const files = await fs.readdir(videoDir);
    const lowerBaseName = baseName.toLowerCase();

    for (const ext of this.THUMBNAIL_EXTENSIONS) {
      const targetFilename = lowerBaseName + ext;
      const match = files.find(file => file.toLowerCase() === targetFilename);
      if (match) {
        return match;
      }
    }
  } catch {
    // Directory read failed, return null
  }
  
  return null;
}
```

**Required change:**

```typescript
private async findThumbnail(videoPath: string): Promise<string | null> {
  const videoDir = path.dirname(videoPath);
  const baseName = path.basename(videoPath, path.extname(videoPath));
  
  const thumbnail = await this.findThumbnailInDirectory(videoDir, baseName);
  if (thumbnail) {
    return thumbnail;
  }

  return this.findThumbnailInDirectory(this.thumbnailsDirectory, baseName);
}

private async findThumbnailInDirectory(directory: string, baseName: string): Promise<string | null> {
  for (const ext of this.THUMBNAIL_EXTENSIONS) {
    const exactThumbPath = path.join(directory, baseName + ext);
    try {
      await fs.access(exactThumbPath);
      return baseName + ext;
    } catch {
      // Exact match failed, continue to case-insensitive fallback
    }
  }

  try {
    const files = await fs.readdir(directory);
    const lowerBaseName = baseName.toLowerCase();

    for (const ext of this.THUMBNAIL_EXTENSIONS) {
      const targetFilename = lowerBaseName + ext;
      const match = files.find(file => file.toLowerCase() === targetFilename);
      if (match) {
        return match;
      }
    }
  } catch {
    // Directory read failed, return null
  }
  
  return null;
}
```

**Why**: Refactor to extract directory search into helper method, then check thumbnails directory as fallback after video directory search fails.

## Edge Cases & Risks

| Risk/Edge Case | Mitigation |
| -------------- | ---------- |
| Thumbnails directory doesn't exist | `fs.readdir()` will throw ENOENT, catch and return null |
| Permission denied on thumbnails directory | Handle gracefully, return null |
| Thumbnail exists in both directories | Video directory takes priority (more specific location) |
| Case sensitivity issues | Already handled by existing case-insensitive fallback logic |
